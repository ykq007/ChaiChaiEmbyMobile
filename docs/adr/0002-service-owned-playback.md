# Make the playback service authoritative

One Media3 `MediaSessionService` owns active playback so playback, progress observation, audio focus, notifications, and system controls survive activity recreation and Adaptive Layout changes. Features submit stable playback intent to a coordinator, which negotiates with Emby and creates an internal playback plan; features do not construct Media3 items, authenticated URLs, headers, or transcoding parameters, accepting a more explicit boundary in exchange for lifecycle continuity and centralized playback policy. The service is the sole source of periodic, transport, background, and terminal playback observations. Once an observation is durably captured, the server platform may deliver or retry that server-user-scoped mutation through WorkManager without keeping the playback service alive; this delivery worker does not own or infer active playback state.
